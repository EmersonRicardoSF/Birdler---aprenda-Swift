O que é MVC?

MVC é uma maneira organizada de criar aplicativos que ajuda os desenvolvedores a manter tudo em ordem e fácil de mexer. Ela divide o aplicativo em três partes:

Model (Modelo):

O que é: Imagina que é um caderno onde você guarda todas as informações importantes.

Exemplo: Se o aplicativo é sobre pessoas, o Modelo guarda informações como nome, idade, etc.

View (Visualização):

O que é: É a tela do aplicativo, o que você vê e com o que você interage.

Exemplo: Botões, textos, imagens, tudo que aparece na tela do seu celular quando você abre o aplicativo.

Controller (Controlador):

O que é: É o cérebro do aplicativo, que faz a ligação entre o caderno de informações (Model) e a tela (View).

Exemplo: Quando você clica em um botão, o Controlador decide o que fazer com essa ação, atualizando as informações e mudando o que aparece na tela.

Como Funciona na Prática?

Vamos imaginar um aplicativo de restaurante que mostra eventos:

  • Interação do Usuário: Você abre o aplicativo e vê uma lista de eventos.
  • Controlador: O Controlador recebe essa ação e pede ao Modelo (caderno) para buscar as informações dos eventos.
  • Modelo: O Modelo pega as informações dos eventos e dá para o Controlador.
  • Controlador: O Controlador pega essas informações e fala para a Visualização (tela) mostrar os eventos para você.

Benefícios:

  • Tudo no Seu Lugar: Como cada parte tem sua função específica, é mais fácil encontrar e corrigir problemas.
  • Fácil de Atualizar: Se precisar mudar alguma coisa na tela, não precisa mexer nas informações guardadas, e vice-versa.
  • Reutilizável: Pode usar as mesmas partes em diferentes partes do aplicativo.

Exemplo Simples:

Vamos pensar em um exemplo básico de um aplicativo de pessoas:

  • Modelo: Um lugar onde guardamos o nome das pessoas.
  • Visualização: A tela que mostra o nome das pessoas.
  • Controlador: O cérebro que decide o que mostrar na tela e como reagir quando você interage.

// Modelo: Guarda o nome de uma pessoa
class Pessoa {
    var nome: String
    init(nome: String) {
        self.nome = nome
    }
}

// Visualização: Mostra o nome da pessoa na tela
class PessoaView {
    func mostrarNome(nome: String) {
        print("O nome é \(nome)")
    }
}

// Controlador: Liga o Modelo e a Visualização
class PessoaController {
    var pessoa: Pessoa
    var pessoaView: PessoaView

    init(pessoa: Pessoa, pessoaView: PessoaView) {
        self.pessoa = pessoa
        self.pessoaView = pessoaView
    }

    func atualizarNome(novoNome: String) {
        pessoa.nome = novoNome
    }

    func mostrarNome() {
        pessoaView.mostrarNome(nome: pessoa.nome)
    }
}

// Usando as partes juntas
let pessoa = Pessoa(nome: "João")
let pessoaView = PessoaView()
let pessoaController = PessoaController(pessoa: pessoa, pessoaView: pessoaView)

pessoaController.mostrarNome() // Mostra: O nome é João
pessoaController.atualizarNome(novoNome: "Maria")
pessoaController.mostrarNome() // Mostra: O nome é Maria

Assim, usando MVC, tudo fica mais organizado e fácil de entender, é uma excelente arquitetura para aqueles que precisam entender os conceitos mais básicos, mas pode não ser a melhor escolha em projetos extremamente grandes, onde existe a necessidade de uma divisão maior.
