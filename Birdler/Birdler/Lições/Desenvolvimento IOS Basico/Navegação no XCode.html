Navegação no Xcode

Navegar no Xcode pode parecer um pouco complexo no início, mas com algumas dicas e truques, você pode se familiarizar rapidamente. Aqui estão alguns pontos principais para ajudar na navegação do Xcode:

Interface Principal do Xcode

NAVIGATOR AREA

Fica no lado esquerdo da tela e contém diversos navegadores:

  • Project Navigator (⌘1): Mostra todos os arquivos do seu projeto.
    • Acesso ao Project Navigator:
    • Clique no ícone de pasta no painel de navegação à esquerda, ou
    • Use o atalho de teclado ⌘1.

    Estrutura do Project Navigator:

    • Arquivos de Código: Arquivos .swift, .m, .h, etc.
    • Arquivos de Interface: Storyboards (.storyboard) e arquivos de Interface Builder (.xib).
    • Recursos: Imagens, arquivos de áudio, vídeos, etc.
    • Configurações do Projeto: O arquivo .xcodeproj ou .xcworkspace.
    • Grupos: Pastas virtuais que você pode usar para organizar seus arquivos.

    Principais Funcionalidades:

    • Adicionar Arquivos: Clique com o botão direito em qualquer lugar na estrutura e selecione "New File..." ou arraste e solte arquivos diretamente no Project Navigator.
    • Organizar Arquivos: Arraste arquivos e pastas para reorganizá-los. Você pode criar novos grupos para manter tudo organizado.
    • Renomear Arquivos: Clique duas vezes no nome do arquivo ou pasta para renomear.
    • Excluir Arquivos: Selecione o arquivo e pressione a tecla Delete. Você pode optar por apenas remover a referência do projeto ou mover o arquivo para o lixo.
    • Pesquisar Arquivos: Use a barra de pesquisa no topo do Project Navigator para encontrar arquivos rapidamente.
    • Ver Detalhes do Arquivo: Selecione um arquivo para ver suas propriedades na Utility Area à direita.

    Atalhos e Truques:

    • Abrir no Editor Principal: Clique uma vez no arquivo.
    • Abrir no Editor Auxiliar: Segure ⌥ (Option) e clique no arquivo.
    • Mostrar no Finder: Clique com o botão direito no arquivo e selecione "Show in Finder".
    • Adicionar ao Target: Clique com o botão direito no arquivo e selecione "Target Membership" para incluir o arquivo em diferentes targets do seu projeto.
    • Agrupar Arquivos por Tipo: Clique com o botão direito no Project Navigator e selecione "Sort by Type".
  • Symbol Navigator (⌘2): Mostra as definições de símbolos do seu projeto (classes, métodos, etc.).
    • Acesso ao Symbol Navigator:
    • Clique no ícone de diagrama de átomos no painel de navegação à esquerda, ou
    • Use o atalho de teclado ⌘2.

    Estrutura do Symbol Navigator:

    • Classes
    • Structs
    • Enums
    • Protocols
    • Extensões
    • Funções
    • Variáveis
    • Constantes

    Principais Funcionalidades:

    • Navegar por Símbolos: Você pode expandir e colapsar itens no Symbol Navigator para ver os detalhes dos símbolos e suas definições.
    • Buscar Símbolos: Use a barra de busca no topo do Symbol Navigator para encontrar rapidamente um símbolo específico.
    • Filtrar Símbolos: Clique no ícone de filtro para escolher quais tipos de símbolos você deseja ver (por exemplo, apenas classes ou funções).
    • Ver Detalhes de Símbolos: Clique em um símbolo para ver sua definição no editor principal.
    • Saltar para Definição: Clique duas vezes em um símbolo para ir diretamente para sua definição no código.
  • Find Navigator (⌘3): Permite buscar texto no seu projeto.
    • Acesso ao Find Navigator:
    • Clique no ícone de lupa no painel de navegação à esquerda, ou
    • Use o atalho de teclado ⌘3.

    Principais Funcionalidades:

    • Buscar Texto: Digite o texto que deseja buscar na barra de busca no topo do Find Navigator.
    • Filtros de Busca: Clique no ícone de filtro ao lado da barra de busca para especificar opções, como case sensitivity, busca em comentários, e uso de expressões regulares.
    • Substituir Texto: Utilize a funcionalidade "Find and Replace" para buscar e substituir texto no projeto.
    • Resultados da Busca: Veja os resultados listados abaixo da barra de busca. Cada resultado mostra o contexto onde o texto foi encontrado.
    • Navegar pelos Resultados: Clique em um resultado para ir diretamente para a linha correspondente no arquivo.
  • Issue Navigator (⌘4): Mostra warnings e erros.
    • Acesso ao Issue Navigator:
    • Clique no ícone de triângulo com ponto de exclamação no painel de navegação à esquerda, ou
    • Use o atalho de teclado ⌘4.

    Principais Funcionalidades:

    • Ver Erros e Avisos: Todos os problemas são listados no Issue Navigator com uma breve descrição.
    • Navegar até o Problema: Clique em um erro ou aviso para ir diretamente para a linha de código correspondente no editor.
    • Detalhes do Problema: Expanda um item no Issue Navigator para ver mais detalhes sobre o problema, incluindo sugestões de correção.
    • Filtrar Problemas: Use a barra de filtros no topo para mostrar apenas erros, apenas avisos, ou ambos.
    • Buscar Problemas: Use a barra de busca no topo para encontrar problemas específicos por mensagem ou arquivo.
  • Test Navigator (⌘5): Mostra testes unitários e de interface.
    • Acesso ao Test Navigator:
    • Clique no ícone de diamante no painel de navegação à esquerda, ou
    • Use o atalho de teclado ⌘5.

    Principais Funcionalidades:

    • Executar Testes: Clique no botão de execução ao lado de um target, classe ou método de teste para executá-lo.
    • Ver Resultados dos Testes: Resultados dos testes são mostrados no painel, indicando quais testes passaram ou falharam.
    • Navegar até o Código de Teste: Clique em uma classe ou método de teste para abrir a implementação no editor principal.
    • Filtrar Testes: Use a barra de busca no topo para encontrar testes específicos.
    • Criar Novos Testes: Clique com o botão direito em um target de teste ou classe e selecione "New File..." para adicionar novos arquivos de teste.
  • Debug Navigator (⌘6): Mostra informações de depuração.
    • Acesso ao Debug Navigator:
    • Clique no ícone de diamante partido com uma linha no painel de navegação à esquerda, ou
    • Use o atalho de teclado ⌘6.

    Principais Funcionalidades:

    • Navegar pelos Threads: Clique em um thread para ver sua pilha de chamadas e estado atual.
    • Ver Detalhes da Pilha de Chamadas: Expanda os frames na pilha de chamadas para ver detalhes sobre cada função chamada.
    • Inspecionar Variáveis: Passe o mouse sobre variáveis na pilha de chamadas para ver seus valores atuais.
    • Adicionar Breakpoints: Clique na margem do editor de código para adicionar breakpoints e pausar a execução do aplicativo em pontos específicos.
    • Continuar, Pausar e Parar a Execução: Use os botões na barra de ferramentas do Xcode para controlar a execução do aplicativo.
    • Step Over, Step Into, Step Out: Use os botões de stepping para navegar pela execução do código linha por linha.
  • Breakpoint Navigator (⌘7): Gerencia breakpoints.
    • Acesso ao Breakpoint Navigator:
    • Clique no ícone de bandeira com um ponto no painel de navegação à esquerda, ou
    • Use o atalho de teclado ⌘7.

    Principais Funcionalidades:

    • Adicionar Breakpoints: Clique na margem esquerda do editor de código ou use o atalho ⌘\ para adicionar um breakpoint na linha atual.
    • Remover Breakpoints: Clique e arraste um breakpoint para fora da margem ou use o Breakpoint Navigator para removê-lo.
    • Ativar/Desativar Breakpoints: Marque ou desmarque a caixa de seleção ao lado de um breakpoint no Breakpoint Navigator para ativá-lo ou desativá-lo.
    • Editar Breakpoints: Clique com o botão direito em um breakpoint e selecione "Edit Breakpoint..." para adicionar condições, ações, e opções de logging.
    • Organizar Breakpoints: Arraste e solte breakpoints no Breakpoint Navigator para organizá-los conforme necessário.
  • Report Navigator (⌘8): Mostra logs de builds e outros relatórios.
    • Acesso ao Report Navigator:
    • Clique no ícone de relógio no painel de navegação à esquerda, ou
    • Use o atalho de teclado ⌘8.

    Principais Funcionalidades:

    • Ver Detalhes de Relatórios: Clique em um item no Report Navigator para ver os detalhes do relatório no painel direito.
    • Filtrar Relatórios: Use a barra de busca no topo para encontrar relatórios específicos.
    • Navegar pelos Erros e Avisos: Relatórios de build e testes mostram erros e avisos que podem ser clicados para navegar diretamente para o código relevante.
    • Exportar Relatórios: Clique com o botão direito em um relatório e selecione "Export" para salvar o relatório para análise externa.
  • Sources Control Navigator (⌘9): Facilita a integração com sistemas de controle de versão como Git.
    • Acesso ao Sources Control Navigator:
    • Clique no ícone de gráfico de ramificação no painel de navegação à esquerda, ou
    • Use o atalho de teclado ⌘9.

    Principais Funcionalidades:

    • Gerenciar Branches: Crie, exclua e alterne entre branches diretamente do Xcode.
    • Fazer Commits: Selecione arquivos modificados e faça commits com mensagens detalhadas.
    • Visualizar Histórico: Veja o histórico de commits e visualize mudanças específicas.
    • Sincronizar com Repositórios Remotos: Puxe e envie mudanças para repositórios remotos.
    • Resolver Conflitos: Use as ferramentas de merge do Xcode para resolver conflitos de forma visual.

Editor Area

Onde você edita seu código. Pode ser dividida em duas partes para comparar arquivos lado a lado (Editor Auxiliar).

  • Visualização de Código: Exibe o código-fonte do arquivo selecionado no Project Navigator.
  • Syntax Highlighting (Realce de Sintaxe): Destaca diferentes partes do código com cores distintas para facilitar a leitura.
  • Autocompletar (Code Completion): Sugere automaticamente opções de conclusão de código conforme você digita.
  • Barra de Ferramentas: Fornece botões e menus para realizar ações como compilar, depurar, executar testes, entre outros.
  • Navegação de Código: Permite navegar rapidamente entre classes, métodos e definições de variáveis.
  • Integração com Interface Builder: Permite alternar entre a visualização de código e a visualização de interface no Interface Builder.

Funcionalidades da Editor Area:

  • Adição de Novos Arquivos: Clique com o botão direito na pasta desejada no Project Navigator e selecione "New File..." para adicionar novos arquivos de código-fonte.
  • Refatoração de Código: Use as funcionalidades de refatoração, como renomear símbolos, extrair métodos, entre outros.
  • Depuração de Código: Configure e gerencie breakpoints diretamente na margem esquerda da Editor Area.
  • Visualização de Documentação: Use o Quick Help Inspector para visualizar documentação embutida diretamente na Editor Area.

Atalhos Úteis:

  • ⌘S: Salva o arquivo atual.
  • ⌘Z: Desfaz a última ação.
  • ⌘⇧Z: Refaz a última ação desfeita.
  • ⌘/ ou ⌘⇧/: Comenta ou descomenta a linha atual ou a seleção de código.

Utility Area

Fica no lado direito e contém:

  • Inspectors: Para configurar propriedades do arquivo selecionado no editor.
    • File Inspector: Exibe informações detalhadas sobre o arquivo selecionado no Project Navigator.
    • Quick Help Inspector: Fornece documentação rápida e informações sobre o símbolo atualmente selecionado na Editor Area.
    • Identity Inspector: Permite configurar as propriedades de identidade de um elemento da interface.
    • Attributes Inspector: Permite ajustar as propriedades visuais de elementos da interface.
    • Size Inspector: Oferece controle sobre o tamanho e a posição de elementos da interface.
    • Connections Inspector: Gerencia as conexões e ligações de saída e entrada entre componentes da interface e o código-fonte.
  • Libraries: Para adicionar novos elementos de interface, código e mídia ao seu projeto.

Toolbar

Na parte superior, onde você pode iniciar o build, rodar o projeto, parar a execução e acessar outras ferramentas.

  • Run (Executar): Botão para compilar e executar o projeto atualmente selecionado. Atalho de teclado: ⌘R.
  • Stop (Parar): Botão para interromper a execução do projeto. Usado para cancelar a execução em curso ou depuração. Atalho de teclado: ⌘..
  • Scheme Selector (Seletor de Esquema): Menu suspenso para selecionar o esquema de compilação e execução atual.
  • Device Selector (Seletor de Dispositivo): Menu suspenso para selecionar o dispositivo ou simulador no qual o aplicativo será executado.
  • Editor Options (Opções do Editor): Botões para alternar entre o Editor de Código, Interface Builder e outras visualizações disponíveis na Editor Area.
  • Debugging Controls (Controles de Depuração): Botões para controlar a execução do aplicativo durante a depuração, como Step Over, Step Into e Step Out.
  • Testing Controls (Controles de Teste): Botões para executar testes unitários e de interface do usuário configurados no projeto.
  • Version Editor (Editor de Versão): Botão para alternar para o Editor de Versão, que permite visualizar e comparar alterações de código ao longo do tempo.

Navegação Rápida

  • Quick Open (⇧⌘O): Abra qualquer arquivo, classe ou símbolo rapidamente digitando o nome.
  • Open Quickly (⇧⌘J): Similar ao Quick Open, mas com foco em símbolos específicos.
  • Jump Bar: No topo do editor, permite navegar rapidamente pela estrutura do código, como classes e métodos.
  • Assistant Editor (⌥⌘↩): Abre um editor adicional ao lado do principal para comparar ou ver arquivos relacionados.

Atalhos de Teclado Úteis

  • Alternar entre editores principais e auxiliares: ⌥⌘↩
  • Abrir rapidamente: ⇧⌘O
  • Construir e rodar o projeto: ⌘R
  • Construir o projeto: ⌘B
  • Parar a execução: ⌘.
  • Adicionar breakpoint: Clique na margem do número da linha ou ⌘\

Depuração

  • Debug Area: Aparece na parte inferior quando você roda o projeto. Mostra variáveis, console e controles de execução.
  • Console: Use para ver logs e mensagens de depuração. Você pode escrever comandos LLDB diretamente aqui.
